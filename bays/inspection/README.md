# Inspection Bay

**Purpose:** Assess the quality of your data and identify problems before you start working with it.

Think of this as putting the data up on a lift — you're looking for anything that needs attention before you take it out on the road.

---

## What's in the notebook

- Missing value counts and percentages per column
- Duplicate row detection
- Data type audit
- Outlier detection (IQR method) for numeric columns
- Value counts for categorical/string columns
- Unique value counts per column

---

## Common Issues You'll Find Here

| Issue | What to do about it |
|---|---|
| Missing values | Head to the Cleaning bay |
| Duplicate rows | Head to the Cleaning bay |
| Wrong data types | Head to the Cleaning bay |
| Outliers | Decide: real data or error? Clean or keep? |
| Unexpected categories | Head to the Cleaning bay for standardization |

---

## When to use this bay

- Right after Intake — before making any changes
- Any time you receive new data from an external source
- Before sharing data or analysis results with others
- When debugging unexpected results in your analysis
